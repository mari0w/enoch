# project-guidelines

用于“二次开发/改造本项目”的规范与约束。只有在需要改造项目时才应用本规范。

## 关键规范
- 控制台必须有足够的日志输出，便于排查问题。
- 日志必须封装为统一模块/组件，不允许散落在业务逻辑里。
- 日志需要支持：
  - 同时输出到控制台与文件。
  - 控制台彩色分级（至少区分 info/warn/error）。
  - 每条日志必须带时间戳。
- 任何执行报错都必须记录日志（包括启动 Codex、调用 Codex 失败、以及 Telegram 请求失败）。
- 日志模块应可配置（开关、文件路径、级别）。
- 改动项目代码时必须补/更新单元测试，并确保单元测试通过后才能提交。

## 使用范围
- 当新增功能、重构结构、或扩展运行流程时，需按本规范补齐日志模块。
- 纯文档或轻微改动不强制引入日志模块。

## 输出要求
- 若涉及日志改造，提交时需说明：
  - 日志输出位置（控制台/文件）
  - 颜色/级别策略
  - 相关配置项
